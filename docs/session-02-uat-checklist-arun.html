📋 How to Use This Checklist
  1. Open the UAT URL above in Google Chrome on your computer or phone.
  2. Before starting each Test Block marked with ⚠ reset: open DevTools → Console and run localStorage.clear(), then reload the page.
  3. For each test: click Pass if the behaviour matches the expectation, or Fail if something goes wrong.
  4. For any Fail, a notes field will appear — describe what you saw (screenshot if possible).
  5. At the end, use the Sign-Off section to approve or flag issues.

Alternative login method for users who prefer not to use passwords
6 tests
⚠ Before this block: run localStorage.clear() in Console and reload.
1
On Sign In screen, click "Prefer a one-time code? Use email OTP →"

Enter your email and click "Send Login Code →"

Enter a wrong 6-digit code (e.g. 000000)

Enter the correct 6-digit code from the email
App loads. Your nickname appears correctly in the header. (For a completely new email, Profile Setup screen appears first.)
5
Click "Didn't get a code? Resend" link
A new code is sent (or a "wait 60 seconds" message appears if too soon). The form stays on the code entry screen — does NOT go back to email entry.
6
Click "← Back" from the OTP email entry screen
Returns to the Sign In / Create Account tabs screen.

G
Loading Splash & Onboarding
No flash of map content during login, first-time onboarding bullets, returning user behaviour
6 tests
1
During sign-in: observe the transition from auth modal to app
After clicking Sign In, you should see: (1) spinner on button → (2) loading splash "Loading your map…" → (3) app map. You should NOT briefly see the map listings panel behind the auth modal.
2
On a fresh account's FIRST login — onboarding splash content
Welcome screen shows 5 bullet points: GPS location, map pins, filter by category/distance, post a listing, save favourites with heart icon. The text is readable (not tiny). Heading says "Welcome to MapIt!"
3
Check that "Tap I'm Interested" is NOT one of the 5 bullets
The onboarding bullets should NOT mention "I'm Interested". There should be exactly 5 bullets, not 6.
4
Tap the splash screen or wait 5 seconds — it dismisses
Tapping the splash dismisses it immediately. If untouched, it auto-dismisses after about 5 seconds. The map is now visible.
5
Sign out and sign back in — onboarding splash does NOT appear again
On second login, the onboarding splash auto-dismisses very quickly (about 0.8 seconds). The 5-bullet welcome screen does NOT appear again for returning users.
6
Clicking a user in "Who's using MapIt?" → no flash of map before splash
When clicking a user button in the Who's using MapIt? screen, the loading splash should appear immediately — you should NOT see the map/sidebar content flash behind the old screen.
